1 What is Regression?

Regression analysis quantifies the relationship between variables. The goal is to model how changes in explanatory variables (X) are associated with changes in the outcome variable (Y).

The linear regression model: \(Y_i = \beta_0 + \beta_1 X_{1i} + \cdots + \beta_k X_{ki} + \varepsilon_i\)

Code 
library(tidyverse)
library(knitr)
library(broom)

# Wage dataset with labour economics flavor
set.seed(2025)
n <- 800
wage_data <- data.frame(
  edu_years = sample(0:20, n, replace=TRUE, prob=dpois(0:20, 9)+0.01),
  experience = sample(0:40, n, replace=TRUE),
  female = rbinom(n, 1, 0.4),
  urban = rbinom(n, 1, 0.55),
  union = rbinom(n, 1, 0.18)
) |>
  mutate(
    log_wage = 4.5 + 0.08*edu_years + 0.04*experience - 0.003*experience^2
               - 0.25*female + 0.15*urban + 0.12*union + rnorm(n, 0, 0.35),
    wage = round(exp(log_wage))
  )

2 Simple Linear Regression

Let’s first explore the education-wage relationship:

Code 
# Scatter with regression line
ggplot(wage_data, aes(x=edu_years, y=wage)) +
  geom_point(alpha=0.3, color="#2563eb", size=1.5) +
  geom_smooth(method="lm", color="#e11d48", se=TRUE, linewidth=1.2,
              fill="#fca5a5", alpha=0.3) +
  scale_y_continuous(labels=scales::label_number(prefix="₹")) +
  labs(title="Years of Education vs Daily Wage",
       subtitle="OLS regression line with 95% confidence band",
       x="Years of Education", y="Daily Wage (₹)") +
  theme_minimal(base_size=11) +
  theme(panel.grid.minor=element_blank())

# Fit simple linear regression
model_simple <- lm(wage ~ edu_years, data=wage_data)

3 Multiple Linear Regression

Now let’s add more variables:

Code 
model_full <- lm(wage ~ edu_years + experience + I(experience^2) + 
                   female + urban + union, data=wage_data)

tidy(model_full) |>
  mutate(
    term = recode(term,
      `(Intercept)`="Intercept",
      edu_years="Education (years)",
      experience="Experience (years)",
      `I(experience^2)`="Experience² (quadratic)",
      female="Female (=1)",
      urban="Urban (=1)",

    significance = case_when(

      TRUE            ~ ""
    )
  ) |>

4 Regression Diagnostics

Always check the four core OLS assumptions:

Code 
par(mfrow=c(2,2), mar=c(4,4,3,1), family="sans")
plot(model_full, which=1:4, 
     col=alpha("#2563eb", 0.4), pch=20, cex=0.7,
     col.smooth="#e11d48", lwd.smooth=2)
Code 
par(mfrow=c(1,1))

What to look for: - Residuals vs Fitted: Should show no pattern (random scatter around 0) - Q-Q Plot: Points should follow the diagonal line (normality of residuals) - Scale-Location: Horizontal band (homoscedasticity — equal variance) - Cook’s Distance: Points above 0.5 are influential observations

5 Coefficient Plot

Code 
tidy(model_full, conf.int=TRUE) |>
  filter(term != "(Intercept)") |>
  mutate(term = recode(term,
    edu_years="Education", experience="Experience",
    `I(experience^2)`="Experience²",
    female="Female", urban="Urban", union="Union")) |>
  ggplot(aes(x=estimate, y=reorder(term, estimate))) +
  geom_vline(xintercept=0, linetype="dashed", color="#94a3b8") +
  geom_errorbarh(aes(xmin=conf.low, xmax=conf.high), height=0.25,

  labs(title="Regression Coefficients with 95% CI",

  theme_minimal(base_size=11) +
  theme(panel.grid.minor=element_blank(),

6 Log-Linear Wage Equation (Mincer Model)

A standard approach in labour economics — using log wages gives percentage interpretations:

Code 
model_log <- lm(log_wage ~ edu_years + experience + I(experience^2) + 
                  female + urban + union, data=wage_data)

tidy(model_log) |>
  filter(term != "(Intercept)") |>
  mutate(
    term = recode(term,
      edu_years="Education (years)",
      experience="Experience",
      `I(experience^2)`="Experience²",
      female="Female (=1)",
      urban="Urban (=1)",
      union="Union (=1)"),
    pct_effect = round((exp(estimate)-1)*100, 2),
    across(c(estimate, std.error, p.value), round, 4)
  ) |>
  select(term, estimate, std.error, p.value, pct_effect) |>
  rename(`% Effect` = pct_effect) |>
  kable(caption="Mincer Wage Equation (Log-Linear): % change in wage per unit")

## Regression Diagnostics

Always check the four core OLS assumptions:

```{r diagnostics}
#| fig-height: 7
#| fig-cap: "Regression diagnostic plots — checking OLS assumptions"

par(mfrow=c(2,2), mar=c(4,4,3,1), family="sans")
plot(model_full, which=1:4, 
     col=alpha("#2563eb", 0.4), pch=20, cex=0.7,
     col.smooth="#e11d48", lwd.smooth=2)
par(mfrow=c(1,1))
```

**What to look for:**
- **Residuals vs Fitted:** Should show no pattern (random scatter around 0)
- **Q-Q Plot:** Points should follow the diagonal line (normality of residuals)
- **Scale-Location:** Horizontal band (homoscedasticity — equal variance)
- **Cook's Distance:** Points above 0.5 are influential observations

## Log-Linear Wage Equation (Mincer Model)

A standard approach in labour economics — using log wages gives percentage interpretations:

```{r mincer}
model_log <- lm(log_wage ~ edu_years + experience + I(experience^2) + 
                  female + urban + union, data=wage_data)

tidy(model_log) |>
  filter(term != "(Intercept)") |>
  mutate(
    term = recode(term,
      edu_years="Education (years)",
      experience="Experience",
      `I(experience^2)`="Experience²",
      female="Female (=1)",
      urban="Urban (=1)",
      union="Union (=1)"),
    pct_effect = round((exp(estimate)-1)*100, 2),
    across(c(estimate, std.error, p.value), round, 4)
  ) |>
  select(term, estimate, std.error, p.value, pct_effect) |>
  rename(`% Effect` = pct_effect) |>
  kable(caption="Mincer Wage Equation (Log-Linear): % change in wage per unit")
```

**Key finding:** The gender wage gap is approximately `r round((1 - exp(coef(model_log)["female"]))*100, 1)`% — women earn about this much less than men after controlling for education, experience, and other factors.
